# Task Manager Frontend

## 📌 Overview

React 기반의 업무 관리 시스템 프론트엔드 애플리케이션입니다.

백엔드 REST API와 연동하여 사용자, 팀, 직급, 권한 관리 기능을 제공하며,  
검색, 페이징, 정렬, 상세 입력 폼 등 실무형 관리 화면 구조를 구현하였습니다.

특히 입력 UX 개선과 데이터 검증을 고려한 구조로 설계하였습니다.

---

## 🛠 Tech Stack

- React
- JavaScript (ES6+)
- Axios

---

## 📂 Structure

- pages: 화면 단위 컴포넌트
- components: 재사용 가능한 UI 컴포넌트
- api: API 통신 모듈
- styles: 공통 스타일 관리
- constants: 공통 옵션 및 유틸성 상수

---

## 🚀 주요 기능

### 사용자 관리
- 사용자 목록 조회
- 사용자 등록 / 수정 / 삭제
- 사용자 권한 선택
- 부서 / 직급 / 상태 정보 관리

### 팀 관리
- 팀 목록 조회
- 팀 등록 / 수정 / 삭제
- 부서장 선택 모달 연동

### 직급 / 권한 관리
- 직급 목록 조회 / 등록 / 수정 / 삭제
- 권한 목록 조회 / 등록 / 수정 / 삭제

### 검색 및 필터링
- 이름, 부서, 직급, 상태 조건 기반 검색
- 서버 API 기반 필터링 처리

### 페이징
- 서버 페이징 데이터 기반 UI 구성

### 정렬
- 정렬 조건 선택 후 조회 시 반영

---

## 🧩 UI/UX 개선

### 목록-상세 연동 구조
- 조회 시 첫 번째 row 자동 선택
- 선택된 row 기반 상세 입력 폼 바인딩
- 선택되지 않은 경우 상세 영역 비활성화 처리

### 입력 흐름 개선
- 신규 추가 시 그리드 상단에 입력용 row 생성
- 상세 입력 값 변경 시 목록에 즉시 반영
- 수정 중 다른 row 선택 시 경고 처리 (데이터 유실 방지)

### 저장 후 상태 유지
- 수정 저장 후 해당 row 재선택
- 신규 등록 후 생성된 데이터 자동 포커스
- 조회/검색 시 상태 초기화 및 첫 row 자동 선택

### 삭제 처리 개선
- 신규 row 삭제 시 서버 호출 없이 화면에서만 제거
- 기존 데이터는 서버 API 호출로 삭제

👉 모든 관리 화면(User, Team, Position, Role)에 동일한 UX 패턴 적용

---

## 📧 이메일 입력 및 검증

- 이메일을 단일 문자열이 아닌 `emailId` / `emailDomain` 구조로 분리하여 관리
- 도메인 선택(gmail.com 등) 또는 직접 입력 방식 지원
- 입력 단계에서 `@` 중복 입력 방지
- 입력값 공백 제거(trim) 후 서버 전송
- 이메일 형식 사전 검증 수행

👉 입력 UX 개선과 데이터 무결성을 동시에 고려한 구조
